# hampter_lab_branding Status

## Summary

`hampter_lab_branding` is the parent project for HAMPTER LAB brand expression. It currently centers on a cold, lab-like web presence that introduces the studio, exposes the TAIGA face runtime, and leaves room for future content and interactive subprojects.

## Current State

- The live public website exists as the main active subproject.
- Brand direction is currently "cold, clean, lab/corporate" rather than playful or doodle-led.
- The homepage now features `TAIGA Face Runtime` as the single active item in `MAKES`.
- Mock `MAKES` and `DEV LOG` entries were removed; the archive is intentionally empty until real posts are published.
- The TAIGA face runtime is integrated into the site at `/taiga/` instead of being deployed as a separate product site.

## Next

- Add real `origin / korean / english` lab notes when the user starts publishing them.
- Expand `MAKES` only with real interactive work rather than placeholder cards.
- Continue recording durable website conventions and deployment facts in the `website` subproject wiki pages.

## Decisions

- Keep project hierarchy explicit: brand parent project, website as a subproject.
- Treat the public site as the main operational surface for HAMPTER LAB branding right now.
- Preserve a monochrome, lab-like tone unless the user explicitly changes the visual direction.
