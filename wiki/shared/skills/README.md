# Shared Skills

Read this page after [shared/index.md](../index.md). This is the shared workflow hub.

Use this folder for conventions that apply across multiple projects.

## Deep Pages

- [wiki-update-workflow.md](./wiki-update-workflow.md) - How to file durable work back into the wiki after a task.
- [project-bootstrap.md](./project-bootstrap.md) - How to add a new project or child project to the wiki structure.
- [public-repo-safety.md](./public-repo-safety.md) - What to redact or avoid before pushing changes to this public repository.

## Rules

- Keep cross-project process here.
- Keep project-specific workflow in the matching project's `skills/README.md`.
- When adding a new shared workflow page, update `shared/index.md` and this file together.

Repository-tracked Codex skill copies can also live outside the wiki when versioning the exact skill files is useful, for example `skills/llm-wiki/SKILL.md`.
